Contexte de l’application

Cette application s’inscrit dans une démarche d’accessibilité numérique et d’exploration des modes de saisie alternatifs. Elle propose un système permettant de saisir du texte à l’aide d’un clavier classique, en reproduisant le fonctionnement d’un clavier braille de type Perkins.

Le braille est un système d’écriture tactile utilisé par les personnes non-voyantes ou malvoyantes. Il repose sur une cellule composée de six points, dont les combinaisons permettent de représenter des lettres, chiffres et symboles. Cependant, l’accès à des périphériques braille physiques comme les claviers Perkins ou les afficheurs braille peut être limité en raison de leur coût ou de leur disponibilité.

L’objectif de cette application est donc de simuler un clavier braille à partir d’un matériel standard (clavier d’ordinateur), afin de :

faciliter l’apprentissage du braille,
proposer une alternative de saisie accessible,
expérimenter des interactions utilisateur adaptées.

Contrairement à une saisie braille classique simultanée (où plusieurs touches sont pressées en même temps), l’application adopte une approche séquentielle. L’utilisateur sélectionne les points un par un, puis valide la lettre. Ce choix permet de :

simplifier l’utilisation sur un clavier standard,
réduire les contraintes techniques liées à la détection multi-touch,
améliorer la robustesse et la fiabilité de la saisie.

L’application intègre également une synthèse vocale basée sur les API Windows (SAPI), permettant de lire le texte saisi. Cette fonctionnalité renforce l’accessibilité en offrant un retour audio immédiat.

🎯 Objectifs
Proposer une interface simple de saisie braille
Adapter le braille à un clavier standard
Expérimenter des interactions accessibles
Fournir un retour vocal pour améliorer l’expérience utilisateur
Créer une base évolutive pour des outils pédagogiques ou d’accessibilité
👤 Public cible
Personnes souhaitant apprendre le braille
Développeurs intéressés par l’accessibilité
Utilisateurs curieux des interfaces alternatives
Projets éducatifs ou expérimentaux
🧩 Positionnement du projet

Ce projet se situe à la croisée de plusieurs domaines :

accessibilité numérique
interfaces homme-machine (IHM)
développement d’applications desktop
adaptation d’un système d’écriture tactile à un environnement numérique

Il peut servir de base pour :

un outil pédagogique,
une application d’assistance,
ou un prototype d’interface inclusive.
